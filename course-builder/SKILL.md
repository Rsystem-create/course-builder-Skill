# course-builder：中文自学教程生成规范

为用户的自学库（如 `~/Documents/Study/Dev/Java/` 下的 Java、Git、Docker、SQL、HTTP、Linux、Go、Python 各套）生成结构完全一致的新教程。**本文件是唯一规范来源，所有条款必须严格遵守。**

## 产出物（四件套）

```
<主题>/
├── docs/                     NN-标题.md，正文分章
├── exercises/                NN-标题.md，与 docs 一一对应的配套练习
├── <主题>完整教程.md          docs 合并单文件（含目录 + 锚点）
└── <主题>练习完整版.md        exercises 合并单文件（含目录 + 锚点）
```

## 强制流程（顺序写死，不许跳步）

1. **先给大纲，等用户批准才动笔。** 大纲列出每章标题和一句话内容说明。用户可能修改方向（例如把「通用 Linux」改成「服务器运维」）——按新方向重写大纲再次征求批准。
2. 写 `docs/01..NN`（每写完一章即保存，不攒批）。
3. 写 `exercises/01..NN`（文件名与 docs 同序号同主题词）。
4. 用 `scripts/merge.py` 生成两个合并单文件。
5. 验证（见下），然后向用户报告：文件树 + 章节表 + 风格规矩如何落实。

## 大纲规则

- **章数不固定**，由主题体量决定（一般 8~15 章，简单主题可更少）。用户批准的大纲有几章就是几章，后续所有校验以此为准。
- 每套教程按叙事弧线编排：基础 → 核心数据/概念 → 该技术的独特气质 → 工程化 → **末章必是串起全书的实战项目**。
- 主题方向要贴合用户实际目标（practical、goal-driven），不是百科全书式罗列。

## docs 章节格式契约

- 文件名：`NN-标题.md`（NN 两位数字，标题中文，可含全角冒号，如 `12-实战：命令行待办工具.md`）。
- 章内结构：
  1. `# 第N章：标题`（N 用汉字：第一章、第十二章）
  2. `> 一句话本章导语`
  3. 引入段落（类比、动机）
  4. 若干正文小节，小节间用 `---` 分隔
  5. `## 小结`：要点 bullet 回顾
  6. 结尾一句「下一章……」指引（末章改为全书收尾语）
- 正文里主动标注**常见坑**（如 ufw 先 allow 22 再 enable、Python 可变默认参数、`w` 模式清空文件等）。

## exercises 章节格式契约

- 文件名与 docs 完全同序号同主题词。
- 结构：
  1. `# 第N章练习：标题`
  2. 链回对应 docs 章节
  3. 提示语：「建议先合上教程，独立完成后再对照末尾参考答案。」
  4. `## 一、概念自测`
  5. `## 二、读码题`（按主题换名：命令行类用「读命令题」、理论类用「理解题」）
  6. `## 三、编程题`（或「动手题」）
  7. `## 参考答案`：**直接写出，绝不折叠在 `<details>` 里**，按题号完整对应。
- 练习文件放 `exercises/` 目录，绝不混进 `docs/`。

## 合并与验证

1. 运行本技能自带脚本（脚本内用绝对路径，不依赖 cwd）：
   ```bash
   python3 <本技能目录>/scripts/merge.py /abs/path/to/<主题> <主题> [教程大标题]
   ```
   第三个参数可选，用于大标题与文件夹名不同的情况（如 Linux 的大标题是「Linux 服务器运维完整教程」）。
2. 验证锚点数 = 大纲章数（**必须加 `-a`**，否则 grep 把含中文的 md 当二进制返回空）：
   ```bash
   grep -a -c '<a id="ch' <主题>完整教程.md <主题>练习完整版.md
   ```
3. 目视检查两个合并文件的 `## 目录`：条目数、章名、锚点连续无跳号。

## 风格规矩（全部强制）

- **全中文**写作，代码注释也用中文。
- **面向新手，不搞太难**；但该讲透的地方（核心概念、易错点）**绝不省笔墨**，宁可多写类比和分步推演。
- **每套教程完全自成体系**：不引用、不依赖库里其他教程。允许「软呼应」（如提一句别的技术怎么做），但概念必须就地讲清。
- 术语首次出现给出英文原词，如「虚拟环境（virtual environment, venv）」。

## 已知坑（历史踩过，必须规避）

- `grep` 数锚点必须 `-a`：含 CJK 的 md 会被当成二进制文件。
- 合并脚本里写死绝对路径；Bash 工具的 cwd 在多次调用间持续存在，靠相对路径曾出过错。
- 临时脚本放会话 scratchpad，不留在用户项目目录里（本技能的 merge.py 是正式脚本，例外）。
- 会话被打断后续写时，先确认哪些文件已存在，从断点继续，不重做已完成的章。
