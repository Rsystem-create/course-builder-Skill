# course-builder

**English** | [中文](#中文版)

A [Claude Code](https://claude.com/claude-code) skill: generate structurally identical **self-study tutorials** for any technical topic, following a fixed "four-piece" spec (chaptered docs + matching exercises + two merged single files with TOC anchors). Language and output format are configured in `config.json` — default **English Markdown**, with Chinese and HTML supported.

## Deliverables

For each topic, the following structure is created in the current directory:

```
<topic>/
├── docs/              NN-title.md, one file per chapter
├── exercises/         NN-title.md, one-to-one with docs
├── merged tutorial    en: <topic>-tutorial.md   zh: <topic>完整教程.md
└── merged exercises   en: <topic>-exercises.md  zh: <topic>练习完整版.md
```

- **docs**: every chapter has an intro line, an opening analogy, body sections, a recap, and a pointer to the next chapter; common pitfalls are flagged proactively.
- **exercises**: concept check + code reading + programming exercises, with solutions written out in full, numbered to match.
- **merged single files**: generated by `scripts/merge.py`, with a TOC and `<a id="chNN">` anchors — handy for straight-through reading or export.
- **optional HTML**: `scripts/to_html.py` additionally produces the two `.html` twins and copies `style.css` into the topic folder — the HTML holds structure only; all styling lives in the external CSS (clickable TOC, dark-mode aware). Just double-click to read in a browser.

## User preferences (config.json)

`course-builder/config.json` stores your defaults, read before generation:

```json
{
  "language": "en",
  "format": "md"
}
```

| Field | Values | Default | Meaning |
|-------|--------|---------|---------|
| `language` | `en` / `zh` | `en` | Tutorial language (body, exercises, file names, TOC headings all follow it) |
| `format` | `md` / `md+html` | `md` | Markdown only, or additionally generate HTML |

Saying "write it in Chinese" or "I want an HTML version" in a single request overrides these temporarily; edit the file to change the long-term default.

## Install

Copy (or symlink) the `course-builder/` directory into Claude Code's skills directory:

```bash
# global (all projects)
cp -r course-builder ~/.claude/skills/

# or current project only
cp -r course-builder .claude/skills/
```

## Usage

In Claude Code, type:

```
/course-builder <topic>
```

e.g. `/course-builder Docker`. **Only this explicit command triggers generation** — mentioning tutorials or courses in natural language never activates the skill.

The workflow is fixed:

1. Claude reads `config.json` to determine language and format.
2. It presents an **outline** first (chapter titles + one-sentence descriptions) and waits for your approval; adjust direction at this step.
3. It writes `docs/` chapter by chapter, then `exercises/`.
4. It runs `scripts/merge.py` to produce the two merged files and self-checks anchor counts (plus `to_html.py` when HTML is wanted).
5. It reports the file tree and chapter table.

## Running the scripts manually

```bash
python3 course-builder/scripts/merge.py /abs/path/to/<topic> <topic> [book title] [--lang en|zh]

# e.g. pass the third argument when the book title differs from the default
python3 course-builder/scripts/merge.py ~/Study/Linux Linux "Linux 服务器运维完整教程" --lang zh

# for the HTML version (same arguments; also copies style.css into the topic folder)
python3 course-builder/scripts/to_html.py ~/Study/Docker Docker
```

When `--lang` is omitted the scripts read `config.json` themselves. Python 3 standard library only.

## Style conventions (summary)

- Body, code comments and exercises all in the chosen language; Chinese tutorials give the English term on first occurrence.
- Beginner-friendly; core concepts and common mistakes are never skimped on.
- Every tutorial is fully self-contained, with no dependencies on other tutorials in your library.
- Chapter count follows the topic's size (usually 8–15); the final chapter is always a hands-on project tying the book together.

Full spec: [`course-builder/SKILL.md`](course-builder/SKILL.md).

# course-builder（中文版）

一个 [Claude Code](https://claude.com/claude-code) 技能（Skill）：按固定「四件套」规范，为任意技术主题生成结构完全一致的**自学教程**（正文分章 + 配套练习 + 两个带目录锚点的合并单文件）。语言与输出格式由 `config.json` 配置，默认**英文 Markdown**，也支持中文与 HTML。

## 产出物

对每个主题，在当前目录下生成如下结构：

```
<主题>/
├── docs/                NN-标题.md，正文分章
├── exercises/           NN-标题.md，与 docs 一一对应的配套练习
├── 合并教程单文件        en: <主题>-tutorial.md   zh: <主题>完整教程.md
└── 合并练习单文件        en: <主题>-exercises.md  zh: <主题>练习完整版.md
```

- **docs**：每章含导语、引入类比、正文小节、小结和下一章指引，主动标注常见坑。
- **exercises**：概念自测 + 读码题 + 编程题，参考答案直接写出、按题号完整对应。
- **合并单文件**：由 `scripts/merge.py` 自动生成，带目录和 `<a id="chNN">` 锚点，方便单文件通读或导出。
- **可选 HTML**：由 `scripts/to_html.py` 额外生成两个同名 `.html`，并复制 `style.css` 到主题目录 —— HTML 只含结构，全部样式在外链 CSS 里（可点击目录、自适配深色模式），双击即可在浏览器阅读。

## 用户偏好（config.json）

`course-builder/config.json` 保存默认偏好，生成前会先读取：

```json
{
  "language": "en",
  "format": "md"
}
```

| 字段 | 取值 | 默认 | 含义 |
|------|------|------|------|
| `language` | `en` / `zh` | `en` | 教程语言（正文、练习、文件名、目录标题全部随之切换） |
| `format` | `md` / `md+html` | `md` | 只出 Markdown，或额外生成 HTML |

单次请求里说「用中文写」「要 HTML 版」可临时覆盖；想改长期默认就改这个文件。

## 安装

将 `course-builder/` 目录复制（或软链）到 Claude Code 的技能目录：

```bash
# 全局安装（所有项目可用）
cp -r course-builder ~/.claude/skills/

# 或仅当前项目
cp -r course-builder .claude/skills/
```

## 使用

在 Claude Code 中输入：

```
/course-builder <主题>
```

例如 `/course-builder Docker`。**只有显式输入这条命令才会触发生成**；自然语言聊到教程/课程不会激活本技能。

流程是固定的：

1. Claude 读取 `config.json` 确定语言和格式。
2. 先给出**大纲**（每章标题 + 一句话说明），等你批准后才动笔；你可以在这一步调整方向。
3. 逐章写 `docs/`，再逐章写 `exercises/`。
4. 运行 `scripts/merge.py` 生成两个合并单文件并自检锚点数（按需再跑 `to_html.py`）。
5. 报告文件树和章节表。

## 手动运行脚本

```bash
python3 course-builder/scripts/merge.py /abs/path/to/<主题> <主题> [教程大标题] [--lang en|zh]

# 例：大标题与默认标题不同时传第三个参数
python3 course-builder/scripts/merge.py ~/Study/Linux Linux "Linux 服务器运维完整教程" --lang zh

# 需要 HTML 版时另跑（参数同 merge.py，会顺带复制 style.css 到主题目录）
python3 course-builder/scripts/to_html.py ~/Study/Docker Docker
```

`--lang` 缺省时脚本自动读 `config.json`。仅依赖 Python 3 标准库。

## 风格约定（摘要）

- 正文、代码注释、练习全部用选定语言；中文教程里术语首次出现附英文原词。
- 面向新手，核心概念和易错点不省笔墨。
- 每套教程完全自成体系，不依赖库里其他教程。
- 章数由主题体量决定（一般 8~15 章），末章必是串起全书的实战项目。

完整规范见 [`course-builder/SKILL.md`](course-builder/SKILL.md)。
